2.1 — Teori: Nmap Scripting Engine (NSE) untuk VA 1 JP TEORI
  • Arsitektur NSE dan kategori script: vuln, exploit, safe, intrusive
  • Cara membaca dan memahami output NSE
  • Menentukan script yang tepat sesuai layanan
  • Korelasi hasil NSE dengan CVE database
Lab 2.2
Nmap NSE — Vulnerability Detection
1.5 JP
bash — Nmap NSE Scripts
# ── Scan semua kategori vuln ──────────────────────────
nmap --script vuln <target>
nmap --script vuln,safe <target>
nmap --script vuln --script-args=unsafe=1 <target>

# ── NSE Script spesifik per layanan ──────────────────
# SMB — EternalBlue (MS17-010)
nmap --script smb-vuln-ms17-010 -p 445 <target>

# SMB — MS08-067
nmap --script smb-vuln-ms08-067 -p 445 <target>

# FTP — vsFTPd Backdoor
nmap --script ftp-vsftpd-backdoor -p 21 <target>

# HTTP — Shellshock
nmap --script http-shellshock -p 80,443 <target>

# SSL — Heartbleed
nmap --script ssl-heartbleed -p 443 <target>

# RDP — BlueKeep
nmap --script rdp-vuln-ms12-020 -p 3389 <target>
Lab 2.2B
Nikto — Web Server Scanner
bash — Nikto
# Scan dasar HTTP
nikto -h http://<target>

# Scan HTTPS
nikto -h <target> -ssl -p 443

# Scan port non-standar
nikto -h <target> -p 8080

# Scan dengan tuning kategori
nikto -h <target> -Tuning 1,2,3,4
# 1=File Upload  2=Misconfiguration  3=Info Disclosure  4=Injection
Lab 2.3
Searchsploit — Validasi Temuan dengan Exploit DB
1.5 JP
bash — Searchsploit
# ── Cari berdasarkan nama & versi ────────────────────
searchsploit vsftpd 2.3.4
searchsploit openssh 7.4
searchsploit apache 2.4.49
searchsploit samba 3.0

# ── Cari dengan filter tipe ──────────────────────────
searchsploit -t remote apache
searchsploit -t webapps php

# ── Baca isi exploit tanpa copy ──────────────────────
searchsploit -x exploits/linux/remote/1234.py

# ── Copy exploit ke direktori kerja ──────────────────
searchsploit -m exploits/linux/remote/1234.py

# ── Update database exploit ───────────────────────────
searchsploit -u
Alur Mapping: Nmap scan (versi service) → Searchsploit (cari exploit) → CVE detail (cek CVSS) → Catat sebagai temuan VA

Lab 3.2
GVM End-to-End Scanning
2 JP
bash — Setup GVM
# ── Inisialisasi (pertama kali saja) ─────────────────
sudo gvm-setup

# ── Jalankan layanan GVM ─────────────────────────────
sudo gvm-start

# ── Cek status instalasi ─────────────────────────────
sudo gvm-check-setup

# ── Akses Web Interface ──────────────────────────────
# URL      : https://127.0.0.1:9392
# Username : admin
Lab 3.2B
Konfigurasi Target & Task di GVM
  1. Buka Configuration → Targets → New Target
    Name: nama target Hosts: IP/range sesuai scope Port List: All IANA TCP & UDP
  2. Buka Scans → Tasks → New Task
    Target: pilih target yg dibuat Scanner: OpenVAS Default Scan Config: Full and Fast
  3. Simpan → klik ▶ Start untuk menjalankan scan
Lab 3.2C
Analisis Hasil Scan GVM
Detail Temuan
  • Summary — deskripsi kerentanan
  • Insight — penjelasan teknis
  • Detection Result — bukti
Filter & Prioritas
  • Filter mulai dari Critical
  • Cek CVSS Score setiap temuan
  • Urutkan berdasarkan severity
Validasi Temuan
  • Cek false positive secara manual
  • Cross-check dengan Nmap
  • Konfirmasi dengan Searchsploit

Lab 4.2
Gobuster & Dirb — Directory & File Enumeration
1 JP
bash — Gobuster
# ── Directory brute force ─────────────────────────────
gobuster dir -u http://<target> -w /usr/share/wordlists/dirb/common.txt

# ── Dengan ekstensi file ──────────────────────────────
gobuster dir -u http://<target> -w /usr/share/wordlists/dirb/common.txt \
          -x php,html,txt,bak

# ── DNS subdomain enumeration ─────────────────────────
gobuster dns -d <domain> -w /usr/share/wordlists/dnsmap.txt

# ── Virtual host scanning ─────────────────────────────
gobuster vhost -u http://<target> -w /usr/share/wordlists/dirb/common.txt
bash — Dirb
# Scan default
dirb http://<target>

# Dengan wordlist medium
dirb http://<target> /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt

# Scan dengan basic authentication
dirb http://<target> -u admin:password
Lab 4.3
WPScan — CMS Vulnerability Scanner (WordPress)
1 JP
bash — WPScan
# ── Scan dasar ───────────────────────────────────────
wpscan --url http://<target>

# ── Enumerate plugin, theme, user ────────────────────
wpscan --url http://<target> --enumerate vp,vt,u
# vp = vulnerable plugins
# vt = vulnerable themes
# u  = enumerate users

# ── Dengan API token (akses CVE lengkap) ──────────────
wpscan --url http://<target> --api-token <TOKEN> --enumerate vp

# ── Deteksi plugin agresif ───────────────────────────
wpscan --url http://<target> --enumerate ap \
       --plugins-detection aggressive
Lab 4.4
SQLMap — SQL Injection Detection
1 JP
bash — SQLMap
# ── Deteksi SQLi pada GET parameter ──────────────────
sqlmap -u "http://<target>/page.php?id=1" --batch
sqlmap -u "http://<target>/page.php?id=1" --dbs --batch

# Naikkan level dan risiko untuk deteksi lebih dalam
sqlmap -u "http://<target>/page.php?id=1" --level=3 --risk=2 --batch

# ── Deteksi SQLi pada form POST ───────────────────────
sqlmap -u "http://<target>/login.php" \
       --data="username=admin&password=test" --batch

# ── Dari file request Burp Suite ──────────────────────
sqlmap -r request.txt --batch --dbs

# ── Deteksi SQLi dengan Cookie ───────────────────────
sqlmap -u "http://<target>/dashboard.php" \
       --cookie="PHPSESSID=abc123; user=admin" --batch --dbs
⚠ Catatan: SQLMap pada fase VA digunakan hanya untuk deteksi kerentanan SQLi, bukan untuk eksploitasi penuh. Gunakan flag --batch untuk menghindari interaksi manual berulang.

Lab 5.1
Enum4linux — SMB/Samba
bash — Enum4linux
# Enumerasi lengkap
enum4linux -a <target>

# Enumerasi spesifik
enum4linux -U <target>  # Users
enum4linux -S <target>  # Shares
enum4linux -P <target>  # Password policy
enum4linux -G <target>  # Groups
enum4linux -o <target>  # OS info

# Verbose mode
enum4linux -v -a <target>
Lab 5.2
SSLScan & SSLyze — SSL/TLS
bash — SSLScan / SSLyze
# SSLScan — cek protokol & cipher
sslscan <target>:443

# SSLyze — analisis mendalam
sslyze <target>:443
sslyze --certinfo --heartbleed --reneg <target>:443

# Yang dicek:
# SSLv2, SSLv3    → deprecated
# TLS 1.0, 1.1    → deprecated
# RC4, DES, 3DES  → cipher lemah
# Heartbleed CVE-2014-0160
# Certificate validity & chain
Lab 5.3
Hydra — Deteksi Weak Authentication / Default Credentials
bash — Hydra
# ── SSH ──────────────────────────────────────────────
hydra -l admin -P /usr/share/wordlists/rockyou.txt ssh://<target>

# ── FTP ──────────────────────────────────────────────
hydra -l admin -P /usr/share/wordlists/rockyou.txt ftp://<target>

# ── HTTP Form POST ────────────────────────────────────
hydra -l admin -P /usr/share/wordlists/rockyou.txt \
  http-post-form "/login.php:username=^USER^&password=^PASS^:Invalid"

# ── RDP ──────────────────────────────────────────────
hydra -l administrator -P /usr/share/wordlists/rockyou.txt rdp://<target>
